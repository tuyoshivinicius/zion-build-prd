# Releases v1.0.0 e v2.0.0 do zion-build-prd — Plano de Implementação

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publicar duas releases no GitHub — `v1.0.0` ancorada em `34c4c61` (o harness que
construiu o `zion-mermaid-editor-app`) e `v2.0.0` no commit de bump do `plugin.json` sobre o
HEAD atual da `main`.

**Architecture:** Não há código a escrever. O trabalho é uma sequência de operações de git e
duas chamadas à API REST do GitHub: tag anotada no commit histórico, commit de bump de versão,
tag anotada no bump, push único com `--follow-tags`, e criação/verificação das releases via
`POST`/`GET` em `/repos/tuyoshivinicius/zion-build-prd/releases`. Cada tarefa termina com um
estado verificável por comando.

**Tech Stack:** git, `curl` contra a API REST do GitHub, token lido de `~/.config/gh/hosts.yml`.

## Global Constraints

- Âncora da v1.0.0: commit **`34c4c61`** (já confirmado como ancestral do HEAD e com
  `"version": "1.0.0"` no `plugin.json` — nenhum commit novo nessa linha).
- O único arquivo alterado pelo bump é `.claude-plugin/plugin.json`, e nele apenas o campo
  `version`: `"1.0.0"` → `"2.0.0"`. `marketplace.json` fica intocado.
- Mensagem exata do commit de bump: `chore(plugin): bump versão 2.0.0`.
- **Nunca** usar `git commit --no-verify` nem `git push --force`. Pre-commit bloqueando ou push
  rejeitado ⇒ parar e reportar.
- Sem branches de release (`release/1.x` está fora de escopo).
- Notas de release em **pt-BR**, tom temático (não é changelog commit-a-commit).
- O `gh` desta máquina é um abridor de browser, **não** o GitHub CLI. Toda interação com a API
  é `curl` com `Authorization: Bearer <token>` lido de `~/.config/gh/hosts.yml`.
- Repositório: `tuyoshivinicius/zion-build-prd`. Branch: `main`.
- Nenhuma mudança em skills, scripts, assets, `docs/prd.md` ou `docs/architecture.md`.

### Task 5: Criar as duas GitHub Releases via API REST

  https://api.github.com/repos/tuyoshivinicius/zion-build-prd | grep -i '^x-oauth-scopes'

Expected: `HTTP/2 200` implícito e um cabeçalho `x-oauth-scopes:` contendo `repo`. Sem `repo`,
